Media Gallery APIs

Centralised asset library supporting upload, metadata, and folder management.

Base URL: /api/v1/gallery

Endpoint directory

MethodPathDescriptionAuth
GET/api/v1/gallery/filesPaginated media browser with folder, owner, uploader, and search filters.Bearer
GET/api/v1/gallery/settingsReturns upload limits and default folders permitted for the requesting user.Bearer
POST/api/v1/gallery/filesUploads one or more files to the gallery, returning metadata for each stored asset.Bearer
PATCH/api/v1/gallery/files/{id}Renames a file, changes its folder, or adjusts ownership notes.Bearer
DELETE/api/v1/gallery/files/{id}Deletes a single file if the caller owns it or has global permissions.Bearer
POST/api/v1/gallery/files/bulk-deleteDeletes multiple files by id in a single operation.Bearer
GET/api/v1/gallery/files/{id}/contentStreams the binary content of the requested gallery item.Bearer
GET/api/v1/gallery/foldersReturns accessible folder tree for the authenticated user.Bearer
POST/api/v1/gallery/foldersCreates a new folder optionally nested inside an existing folder.Bearer
PATCH/api/v1/gallery/folders/{id}Renames an existing folder.Bearer
DELETE/api/v1/gallery/folders/{id}Deletes a folder when empty or when cascading delete is permitted.Bearer

List files GET /api/v1/gallery/files

Paginated media browser with folder, owner, uploader, and search filters.

Parameters

NameDescription
pagePage index
sizePage size
folderIdFilter by folder
searchFilename search

Success response

HTTP 200

{
  "content": [
    {
      "id": 901,
      "filename": "aurora-primary.jpg",
      "url": "https://cdn.example.com/gallery/aurora-primary.jpg",
      "sizeBytes": 245678
    }
  ],
  "page": 0,
  "size": 20,
  "totalElements": 1,
  "totalPages": 1,
  "last": true
}

Error scenarios

No specific error payloads documented.

Upload files POST /api/v1/gallery/files

Uploads one or more files to the gallery, returning metadata for each stored asset.

Headers

NameValue
Content-Typemultipart/form-data

Request body

No body required.

Success response

HTTP 200

[
  {
    "id": 902,
    "filename": "detail.jpg",
    "url": "https://cdn.example.com/gallery/detail.jpg",
    "sizeBytes": 198765
  }
]

Error scenarios

StatusDescriptionExample
HTTP 400No files provided
{
  "error": "Bad Request",
  "message": "FILES_REQUIRED"
}
