Platform Settings APIs

Storefront configuration, email settings, and template management.

Base URL: /api/v1/settings

Endpoint directory

MethodPathDescriptionAuth
GET/api/v1/settingsReturns global store configuration including currency and contact details.Bearer
PATCH/api/v1/settingsUpdates general settings fields supplied in the request body.Bearer
GET/api/v1/settings/themeReturns the storefront theme tokens for anonymous rendering.Public
GET/api/v1/settings/emailReturns SMTP or transactional email configuration.Bearer
POST/api/v1/settings/email/testSends a test email using the current settings to validate connectivity.Bearer
GET/api/v1/settings/email/templatesReturns available system email templates grouped by category.Bearer
GET/api/v1/settings/email/templates/{id}Loads template subject and body for editing.Bearer
PUT/api/v1/settings/email/templates/{id}Updates template subject, HTML, and plain text body.Bearer

Get settings GET /api/v1/settings

Returns global store configuration including currency and contact details.

Success response

HTTP 200

{
  "storeName": "Glow Commerce",
  "currency": "USD",
  "supportEmail": "support@example.com",
  "maintenanceMode": false
}

Error scenarios

No specific error payloads documented.

Update settings PATCH /api/v1/settings

Updates general settings fields supplied in the request body.

Headers

NameValue
Content-Typeapplication/json

  "maintenanceMode": false

{

Send test email POST /api/v1/settings/email/test

Sends a test email using the current settings to validate connectivity.

Headers

NameValue
Content-Typeapplication/json

Request body

{
  "to": "qa@example.com"
}

Success response

HTTP 200

{
  "delivered": true,
  "messageId": "20250115-abc"
}

Error scenarios

StatusDescriptionExample
HTTP 400SMTP failure

  "error": "Bad Request",
  "message": "SMTP_AUTH_FAILED"
