Role APIs

Manage reusable role definitions and bind permissions for RBAC.

Base URL: /api/v1/roles

Endpoint directory

MethodPathDescriptionAuth
GET/api/v1/rolesReturns paginated roles with permission counts.Bearer
POST/api/v1/rolesCreates a role skeleton optionally seeded with permissions.Bearer
GET/api/v1/roles/{id}Fetches full role metadata with permission IDs.Bearer
PUT/api/v1/roles/{id}Updates role name/description or slug.Bearer
DELETE/api/v1/roles/{id}Removes a role when no users depend on it.Bearer
POST/api/v1/roles/{id}/permissionsAdds permission IDs to the role.Bearer
DELETE/api/v1/roles/{id}/permissions/{permissionId}Removes a permission from a role.Bearer

List roles GET /api/v1/roles

Returns paginated roles with permission counts.

Parameters

NameDescription
pagePage index
sizePage size
sortSort field
directionasc/desc

Success response

HTTP 200

{
  "content": [
    {
      "id": 3,
      "name": "Catalog Manager",
      "code": "CATALOG_MANAGER",
      "permissions": 12
    }
  ],
  "page": 0,
  "size": 20,
  "totalElements": 1,
  "totalPages": 1,
  "last": true
}
