Categories & Brands APIs

Manage taxonomy trees, category media, and brand registries for the storefront.

Base URL: /api/v1

Endpoint directory

MethodPathDescriptionAuth
GET/api/v1/categoriesReturns paginated categories including hierarchy metadata.Bearer
GET/api/v1/categories/{id}Retrieves category details including SEO and content blocks.Bearer
GET/api/v1/categories/optionsLightweight category list for dropdowns, excluding an optional id to prevent cycles.Bearer
POST/api/v1/categoriesCreates a new category with optional parent linkage and SEO metadata.Bearer
PUT/api/v1/categories/{id}Updates category content, SEO data, or parent assignment.Bearer
DELETE/api/v1/categories/{id}Deletes a category if no products reference it.Bearer
POST/api/v1/categories/assets/{type}Uploads a category icon, banner, or cover image (type path segment).Bearer
GET/api/v1/categories/assets/{type}/{key}Streams a stored category asset by key.Public
GET/api/v1/brandsPaginated brand directory supporting search.Bearer
POST/api/v1/brandsCreates a brand record optionally linking to a media logo.Bearer
POST/api/v1/brands/assetsUploads and stores a brand logo image.Bearer
GET/api/v1/brands/assets/{key}Streams a stored brand logo.Public

List categories GET /api/v1/categories

Returns paginated categories including hierarchy metadata.

Parameters

NameDescription
pagePage index
sizePage size
searchOptional search

Success response

HTTP 200

{
  "content": [
    {
      "id": 3,
      "name": "Lighting",
      "slug": "lighting",
      "parent": null,
      "status": "PUBLISHED"
    }
  ],
  "page": 0,
  "size": 20,
  "totalElements": 1,
  "totalPages": 1,
  "last": true
}

Error scenarios

No specific error payloads documented.

Get category GET /api/v1/categories/{id}

  "name": "Lighting",
  "slug": "lighting",
  "parentId": null,
  "seo": {
    "title": "Lighting"
  }

StatusDescriptionExample

  "message": "CATEGORY_NOT_FOUND"
}

Create category POST /api/v1/categories

Creates a new category with optional parent linkage and SEO metadata.

Headers

NameValue
Content-Typeapplication/json

Request body

{
  "name": "Smart Lighting",
  "slug": "smart-lighting",
  "parentId": 3,
  "description": "App controlled fixtures"
}

Success response

HTTP 201

{
  "id": 11,
  "slug": "smart-lighting"
}

Error scenarios

StatusDescriptionExample
HTTP 409Slug exists
{
  "error": "Conflict",
  "message": "CATEGORY_SLUG_EXISTS"
}
