User Management APIs

CRUD, status controls, role assignment, and profile updates for internal and customer accounts.

Base URL: /api/v1

Endpoint directory

MethodPathDescriptionAuth
GET/api/v1/usersReturns a paginated list of users filtered by optional search, sort, and paging parameters.Bearer
GET/api/v1/users/summaryAggregated counts of active, invited, and locked users for dashboard widgets.Bearer
POST/api/v1/usersCreates a new user and dispatches an activation email if enabled.Bearer
GET/api/v1/users/{id}Fetches a full user profile including assigned roles and direct permissions.Bearer
PUT/api/v1/users/{id}Overwrites core profile details including contact information and roles.Bearer
PATCH/api/v1/users/{id}/statusActivates, suspends, or locks a user account.Bearer
DELETE/api/v1/users/{id}Removes a user. Audit history retains their activities.Bearer
POST/api/v1/users/{id}/rolesAdds one or more role IDs to the user, merging with existing assignments.Bearer
DELETE/api/v1/users/{userId}/roles/{roleId}Detaches a role from the user without affecting other assignments.Bearer
PUT/api/v1/users/{id}/permissionsSets explicit allow/deny permissions overriding role defaults.Bearer
POST/api/v1/users/{id}/verifyMarks a user's email as verified without email interaction (admin only).Bearer
POST/api/v1/users/{id}/unlockClears login lock flags so the user can sign in again.Bearer
PUT/api/v1/profileAllows the authenticated user to change their name, avatar, or contact number.Bearer

List users GET /api/v1/users

Returns a paginated list of users filtered by optional search, sort, and paging parameters.

Parameters

NameDescription
searchOptional name/email fragment
pageZero-based page index
sizePage size
sortSort field
directionasc or desc

Success response

HTTP 200

{
  "content": [
    {
      "id": 12,
      "email": "ops@example.com",
      "firstName": "Operations",
      "lastName": "Lead",
      "roles": [
        "ADMIN"
      ],
      "status": "ACTIVE"
    }
  ],
  "page": 0,
  "size": 20,
  "totalElements": 1,
  "totalPages": 1,
  "last": true
}

Error scenarios

StatusDescriptionExample
HTTP 403Missing permission
{
  "error": "Forbidden",
  "message": "USER_VIEW_DENIED"
}

Create user POST /api/v1/users

Creates a new user and dispatches an activation email if enabled.

Headers

NameValue
Content-Typeapplication/json

Request body

{
  "email": "new.manager@example.com",
  "firstName": "New",
  "lastName": "Manager",
  "password": "Temp@123",
  "roles": [
    1
  ],
  "sendInvite": true
}

Success response

HTTP 201

{
  "id": 89,
  "email": "new.manager@example.com",
  "status": "INVITED"
}

StatusDescriptionExample
HTTP 409Email already exists
{
  "error": "Conflict",
  "message": "EMAIL_IN_USE"
